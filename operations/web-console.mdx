The Druid web console provides a comprehensive UI for managing your cluster, loading data, executing queries, and monitoring system health.

<Info>
The web console is hosted by the [Router](/design/router) service and accessed at `http://<ROUTER_IP>:<ROUTER_PORT>`.
</Info>

## Prerequisites

The web console requires the following configurations (enabled by default):

<Steps>
  <Step title="Router Management Proxy">
    Enable the [Router's management proxy](/design/router#enable-the-management-proxy).
  </Step>
  <Step title="Druid SQL">
    Enable [Druid SQL](/configuration#sql) for Broker processes.
  </Step>
</Steps>

```properties
# Typically already enabled in default configurations
druid.router.managementProxy.enabled=true
druid.sql.enable=true
```

<Warning>
**Security Note:** Without proper [authentication and authorization](/operations/security-overview) configured, any user accessing the web console has the same privileges as the OS user running Druid. Always configure security for production environments.
</Warning>

## Accessing the Web Console

Navigate to the Router's address in your web browser:

<CodeGroup>
```bash HTTP
http://<ROUTER_IP>:8888
```

```bash HTTPS (with TLS)
https://<ROUTER_IP>:9088
```
</CodeGroup>

<Tip>
If you've enabled authentication, you'll be prompted to log in with your credentials.
</Tip>

## Query View

The enhanced Query view supports multi-stage query execution with SQL-based ingestion.

### Key Features

<Tabs>
  <Tab title="Multi-Tab Interface">
    - Open multiple query tabs simultaneously
    - Each tab maintains its own query context
    - Click **+** to open a new tab
    - Right-click tab names to rename, duplicate, or close
  </Tab>
  
  <Tab title="Schema Browser">
    - Browse available schemas and datasources
    - View table columns and types
    - Drag-and-drop tables into queries
  </Tab>
  
  <Tab title="External Data">
    Click **Connect external data** to:
    - Sample external data sources
    - Generate EXTERN function definitions
    - Create initial queries for data exploration
  </Tab>
</Tabs>

### Query Execution

<Steps>
  <Step title="Write Your Query">
    ```sql
    SELECT
      __time,
      COUNT(*) AS events
    FROM wikipedia
    WHERE __time >= CURRENT_TIMESTAMP - INTERVAL '1' HOUR
    GROUP BY 1
    ORDER BY 1 DESC
    ```
  </Step>
  
  <Step title="Select Engine">
    Choose the query engine from the dropdown:
    - **Auto** (recommended): Automatically selects appropriate engine
    - **SQL Native**: Traditional Druid SQL
    - **SQL MSQ Task**: Multi-stage query engine for ingestion and complex queries
  </Step>
  
  <Step title="Run or Preview">
    - **Run**: Execute the full query
    - **Preview**: For INSERT/REPLACE queries, runs without inserting and with a LIMIT
  </Step>
</Steps>

### Live Query Reports

When using the MSQ engine, the console displays real-time query progress:

<Accordion title="Progress Indicators">
- **Overall Progress**: Main progress bar showing total completion
- **Current Stage**: Progress of the currently executing stage
- **Stage Details**: Expand each stage to see:
  - Worker statistics
  - Partition-level metrics
  - Rows processed and output
  - Memory usage
</Accordion>

## Datasources View

Manage all datasources in your cluster from a centralized view.

### Features

<CardGroup cols={2}>
  <Card title="Datasource List" icon="table">
    View all datasources with:
    - Total size
    - Segment count
    - Availability percentage
    - Row count estimates
  </Card>
  
  <Card title="Segment Timeline" icon="chart-gantt">
    Toggle **Show segment timeline** to visualize:
    - Time-based segment distribution
    - Segment gaps
    - Replication status
  </Card>
  
  <Card title="Retention Rules" icon="calendar">
    Configure rules to control:
    - Data retention periods
    - Segment replication
    - Historical tier assignments
  </Card>
  
  <Card title="Compaction" icon="compress">
    Enable and configure:
    - Automatic compaction
    - Compaction schedules
    - Segment size optimization
  </Card>
</CardGroup>

<Frame>
  <img src="/assets/web-console-04-datasources.png" alt="Datasources view" />
</Frame>

### Actions

Right-click any datasource to:

- **Query**: Open in Query view
- **Edit retention rules**: Modify load/drop rules
- **Configure compaction**: Set up automatic compaction
- **Drop datasource**: Permanently delete
- **Mark as unused**: Soft delete (can be restored)

<Warning>
Dropping a datasource permanently deletes all segments and metadata. This action cannot be undone.
</Warning>

## Lookups View

Manage query-time lookup tables for dimension value enrichment.

<Frame>
  <img src="/assets/web-console-13-lookups.png" alt="Lookups view" />
</Frame>

### Creating Lookups

<Steps>
  <Step title="Open Lookups">
    Access from **Home** view or top navigation **⋮** menu
  </Step>
  
  <Step title="Create Lookup">
    Click **Add lookup** and configure:
    - Name
    - Version
    - Type (map, JDBC, etc.)
    - Lookup data
  </Step>
  
  <Step title="Apply">
    Lookups are loaded into Broker and Historical memory
  </Step>
</Steps>

### Using Lookups in Queries

```sql
SELECT
  LOOKUP(country_code, 'country_names') AS country,
  COUNT(*) AS events
FROM events
GROUP BY 1
```

## Tips and Tricks

<AccordionGroup>
  <Accordion title="View Underlying SQL">
    Most views are powered by Druid SQL. Click **⋮** → **View SQL query for table** to see and customize the underlying query.
  </Accordion>
  
  <Accordion title="Keyboard Shortcuts">
    In the Query view:
    - `Ctrl/Cmd + Enter`: Run query
    - `Ctrl/Cmd + K`: Open command palette
    - `Ctrl/Cmd + /`: Toggle comments
  </Accordion>
  
  <Accordion title="Export Results">
    Query results can be exported as:
    - CSV
    - TSV
    - JSON
  </Accordion>
  
  <Accordion title="Dark Mode">
    Toggle dark mode from the user menu in the top-right corner.
  </Accordion>
</AccordionGroup>
